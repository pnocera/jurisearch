# Phase 0 Code Review

## Findings

1. **`eval france-legi` is advertised as one-shot-only, but the session dispatcher reports it as an unknown command.**  
   `COMMANDS` now registers `eval france-legi` as an implemented command (`crates/jurisearch-core/src/contract.rs:158-164`), and `SESSION_EXCLUDED_COMMANDS` marks that exact name as one-shot-only (`crates/jurisearch-core/src/contract.rs:188-192`). `agent_help()` therefore tells agents the command exists and is not available over the session protocol (`crates/jurisearch-core/src/contract.rs:212-219`). The actual session dispatcher, however, only returns `not_implemented` for `"related" | "ingest" | "eval" | "sync"` (`crates/jurisearch-cli/src/main.rs:4253-4268`). A session request with `"command":"eval france-legi"` falls through to `bad_input("unknown session command ...")`, which contradicts the contract and the Phase 0 session-parity truth requirement. The new test misses this because it hard-codes `["related", "ingest", "eval", "sync"]` instead of iterating `SESSION_EXCLUDED_COMMANDS` (`crates/jurisearch-cli/src/main.rs:6079-6085`).  
   **Fix:** make `dispatch_session_request` reject `"eval france-legi"` with `ErrorObject::not_implemented`, remove or justify the non-contract `"eval"` session spelling, and change the invariant test to iterate `SESSION_EXCLUDED_COMMANDS` so this table cannot drift again. Ideally also assert that every contract command for which `command_session_available(name)` is true has a routed session arm or a documented protocol exception.

2. **`EvalFranceLegiResponse` schema does not describe the JSON that `eval france-legi` emits.**  
   The added schema says `schema_version` is a string and documents a `command: "eval france-legi"` property (`crates/jurisearch-core/src/schema.rs:650-659`). The actual artifact emitted by `eval_france_legi_payload()` via `france_legi_artifact()` has `schema_version: 1` as a number, no `command` field, and important top-level fields such as `kind`, `jurisdiction`, `claim_scope`, `source`, `retriever`, `embedding`, `evidence`, and `reason` (`crates/jurisearch-cli/src/main.rs:864-930`). That makes `help schema --json` untruthful for the newly registered command and violates the Phase 0 rule that newly emitted response fields must be schema'd in the same change.  
   **Fix:** either update the emitted artifact to match the schema or, preferably, update `EvalFranceLegiResponse` to match the existing artifact shape: `schema_version` as integer/const `1`, `kind` const `phase1_france_legi_benchmark`, `state`, `jurisdiction`, `claim_scope`, `source`, `retriever`, `embedding`, `thresholds`, `categories`, `provenance`, `evidence`, and `reason`. Add a focused unit test that builds a sample `france_legi_artifact()` and checks its top-level keys/types against the schema body.

3. **The Phase 0 help-text pass is incomplete and lacks the required full command-matrix guard.**  
   T0.1 required every command path to have rich help/long-about text plus at least one example, and required a snapshot-style test over the full command matrix. Several top-level paths still only have one-line descriptions with no example or long-form guidance, including `status`, `model`, `setup`, `session`, `batch`, `ingest`, `eval`, `sync`, and `help` (`crates/jurisearch-cli/src/main.rs:153-170`). The nested ingest flags also still have no field help for `plan-archives --source/--archives-dir` and most `legi-archives` flags (`crates/jurisearch-cli/src/main.rs:417-439`). I also did not find the required full-matrix help snapshot/invariant in `crates/jurisearch-cli/tests/cli_contract.rs`; existing help tests only smoke-test `help agent` / `help schema --json` (`crates/jurisearch-cli/tests/cli_contract.rs:53-224`).  
   **Fix:** complete the help pass for every enumerated command path in T0.1, including ingest subcommand args and the global `--index-dir` flag, then add the full command-matrix test that asserts each command has non-empty long help and each argument has help text. If any path intentionally remains terse, put it in an explicit allow-list with the reason.
