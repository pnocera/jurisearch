## Findings

- **BLOCKER** `apps/dashboard/shared/src/dto.ts:198` parses the real lifecycle journal fixture to `unit: "init.scope"` because `_SYSTEMD_UNIT` wins over `UNIT`. On CT-111 lifecycle lines are exactly the sparse/common case and carry the producer service in `UNIT` while `_SYSTEMD_UNIT` is `init.scope`; with `/api/logs` returning multiple groups, M4 cannot attribute those lines to `jurisearch-producer-<group>.service`. The tests currently lock in the bad value at `apps/dashboard/shared/src/dto.test.ts:118` and `apps/dashboard/server/src/providers/parse.test.ts:105`. Fix by selecting the first candidate among `UNIT` and `_SYSTEMD_UNIT` that matches `jurisearch-producer-*.service` (falling back to `_SYSTEMD_UNIT ?? UNIT` only when neither matches), then update the fixture assertions to expect the producer service unit for the lifecycle line.

- **WARN** `PackageEntryDTO` is still unexercised for the future non-empty manifest path. The real fixture has `payload.packages: []`, and the provider/parser tests only assert that empty packages are tolerated (`apps/dashboard/server/src/providers/parse.test.ts:81`, `apps/dashboard/server/src/providers/providers.test.ts:166`). The schema at `apps/dashboard/shared/src/dto.ts:142` appears compatible with `RemotePackageEntry`, but a rename or missing required field would stay false-green until the first increment is published. Fix by adding a synthetic signed manifest with one `RemotePackageEntry` shaped from `crates/jurisearch-package/src/manifest/remote.rs:79` and asserting `packageId`, `fromSequence`, `toSequence`, `rowCounts`, `schemaVersion`, `embeddingFingerprint`, `sha256`, and `signature`.

- **NIT** The read-only audit is useful but easy to game. `apps/dashboard/server/src/adapters/adapters.test.ts:13` checks only public method names, so it would still pass if `FileAdapter.read()` gained a write internally or if a provider sent a mutating command through `ProcessRunner.run()`. Current source stays read-only: file I/O is `readFile`/`stat`/`readdir`, and provider commands are `status`, `journalctl`, and `systemctl list-timers`. Fix by adding a narrow source audit for forbidden `node:fs/promises` imports and provider command verbs, or by asserting every exported command builder against an allowlist of exact read-only argv prefixes.
