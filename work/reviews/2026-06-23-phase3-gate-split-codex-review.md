# Phase III Gate Split Review

Reviewed commit `9d72997` against `/tmp/codex-phase3-gate-split-review.md`.

## Findings

- **BLOCKER:** The status gate accepts a `passed` France-LEGI artifact without the routing-backend audit that the split relies on as proof that the two gating categories were resolved structurally. The runner records the live backend faithfully: `france_legi_search_documents` reads `response["routing"]["chosen_backend"]` after `search_with_postgres` writes the production routing object, and the eval loops aggregate that value per category. But `phase1_france_legi_artifact_errors` only calls `phase1_france_legi_validate_category`, and that helper validates threshold, metric value, and query count only; it never requires `categories.structured_citation_resolution.routing_backends.structured_citation` or `categories.temporal_version_pinning.routing_backends.structured_citation` to exist or to cover the recorded query count. The current `valid_france_legi_artifact()` test fixture omits `routing_backends` entirely, yet `france_legi_payload_consumes_valid_artifact` and the Phase 1 gate test still pass/open the claim. A hand-authored artifact can therefore set high structured metrics with no structural-routing proof, or even include a contradictory `routing_backends: {"hybrid": ...}`, and still pass the release gate. This breaks the gate-integrity requirement that the split prove structured categories were resolved by the input-driven structured resolver, not by an answer-aware or fuzzy harness. Require the routing audit for all categories before accepting a `passed` artifact; at minimum, the two gating categories should require `routing_backends.structured_citation == queries` (or an explicitly justified equivalent), and semantic should require the expected advisory backend accounting.

- **HIGH:** Runner state and status re-derivation can diverge at threshold boundaries because the runner decides `state` from raw floating-point metrics, but records only `round_metric(... )` to three decimals, and `phase1_france_legi_validate_category` re-derives validity from the rounded `categories.*.metric_value`. For example, with a custom qrel cap, a raw structured metric just below the floor such as `0.9496` would make `france_legi_artifact` write `state: "failed"` while recording `metric_value: 0.95`; the status validator would see the rounded value as satisfying the `0.95` threshold. The unmodified runner artifact remains blocked because `phase1_france_legi_payload_with_path` preserves `state: "failed"`, but changing only the state field to `passed` would let the status gate accept an artifact that the runner itself failed. That undercuts the requested independent re-derivation. Record enough precision for gate metrics, or round/floor in a way that cannot turn a below-floor raw metric into a passing recorded metric, and add a boundary test.

## Verified Behavior

- The two structured categories are hardcoded as gating in `phase1_france_legi_artifact_errors`; the advisory flag is only passed for `semantic_retrieval`, so marking a structured category as advisory in artifact JSON does not bypass the metric/threshold checks.
- For gating categories, `phase1_france_legi_validate_category` requires the recorded threshold to be at least the policy floor and requires the recorded metric to meet the recorded threshold. A hand-authored artifact cannot lower `structured_citation_recall_at_10_min` below `0.95` or `temporal_version_exactness_at_10_min` below `0.90` without an error.
- The semantic category is correctly non-gating for metric value: it must record a threshold, metric, and minimum query count, but its `0.116` metric does not produce a gate error and does not affect runner `state` or `claim_allowed`.
- The production routing audit source is faithful in the runner: `search_with_postgres` computes `chosen_backend` from citation-intent routing or hybrid fallback and writes it to `response["routing"]["chosen_backend"]`; `france_legi_search_documents` returns that exact field, and the runner aggregates it into each category's `routing_backends`.

VERDICT: FIXES_REQUIRED
